# Tailwind CSS v4 規約

## 基本方針

- スタイルは Tailwind のユーティリティクラスで組む。独自 CSS は原則書かない
- 装飾・レイアウトは可能な限り Tailwind の標準機能で再現する。素の CSS / カスタムプロパティ / `@apply` は、ユーティリティで表現できない場合の最後の手段とする
- オーソドックスな実装を優先し、凝った独自表現より標準的なパターンを選ぶ

## バージョンと構成

- Tailwind CSS は v4 系を使う。`@tailwindcss/vite` プラグイン経由で読み込む
- v3 以前の `tailwind.config.js` 中心の構成は使わない
- グローバル CSS のエントリで `@import "tailwindcss";` を記述する

## 値の選び方

- 色・余白・フォントサイズ・角丸などは Tailwind のデフォルトスケール (`p-4`, `text-lg`, `rounded-md` 等) を使う
- 任意値 (`w-[473px]`, `text-[13.5px]` のような `[...]` 記法) は濫用しない。半端な独自値を作らず、近い標準値に寄せる
- デザイン上どうしても必要な場合のみ任意値を使い、繰り返すなら次の「テーマ拡張」でトークン化する

## テーマ拡張

- 追加するトークンは必要最小限にとどめる。ブランドカラーや指定フォントなど、デザイン上必須のものだけを `@theme` で定義する
- カラー・フォント・ブレークポイント等のトークンは CSS の `@theme` ブロックで定義する (JavaScript 設定ファイルは使わない)
- 細かい一回限りの値を `@theme` に増やさない。スケールは Tailwind 標準を基準に保つ

## クラスの記述

- クラスの並び順は `prettier-plugin-tailwindcss` に委譲する (手動で並べ替えない)
- 同じ組み合わせを繰り返す場合はコンポーネント化を優先し、`@apply` の多用は避ける
